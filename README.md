# EMSHOP-官方系统源码发布

## 功能简介
EMSHOP 发卡系统是一套开源的数字化商品销售解决方案，专为虚拟商品交易设计。系统具备以下核心功能：

## 核心功能
- **多种商品类型**：支持多种商品类型，每种商品类型都有独立的且适配该类型的库存管理和发货模式
- **分站功能**：客户可自助开通分站销售主站商品，可自主切换分站模板
- **SKU多规格**：企业级的多规格商品功能，轻松实现多种商品规格
- **博客功能**：系统内部封装完整的博客功能，利于全站SEO优化
- **应用商店**：模板、支付、通知、美化、客服、商品类型、其他系统扩展功能 丰富的应用功能，应有尽有
- **系统配置**：更多更全面的系统配置，尽全力满足用户的各种需求

## 技术架构
- 基于原生 PHP 开发，未使用任何框架，保障程序安全及运行速度的同时兼容性强，易于部署，轻松扩展
- MySQL 数据库存储，保证数据安全可靠
- 插件化设计，便于二次开发和功能扩展

## 1. 安装前环境准备
本系统在以下环境中开发，其他版本仅作少量测试。如您在其他环境中发现问题，请向我们提交反馈。

- PHP 7.4
- MySQL 5.6
- Nginx 1.24 ~ 1.26

**预留端口**：80（HTTP）、443（HTTPS）、3306（MySQL），确保端口未被防火墙拦截

## 2. 宝塔面板创建站点
1. 进入宝塔面板「网站」→「添加站点」
2. 域名：填写你的域名（如`www.example.com`，无域名可填服务器 IP）
3. 数据库：选择「创建数据库」，填写数据库名、用户名、密码（务必记录，后续安装需用到）
4. PHP 版本：选择步骤 1 中安装的对应 PHP 版本
5. 点击「提交」，站点创建完成

## 3. 设置伪静态
1. 进入宝塔面板「网站」
2. 点击刚刚创建的网站名或点击设置，选择伪静态。填入伪静态内容，点击保存即可。

```nginx
location / {
    index index.php index.html;
    if (!-e $request_filename){
        rewrite ^/(.*)$ /index.php last;
    }
}
```

## 4. 源码安装
1. 进入宝塔面板「网站」，点击网站根目录，进入网站根目录。
2. 点击「上传」→「选择文件」，选择本地的 EMSHOP 源码压缩包，点击开始上传，等待上传完成。
3. 上传完成后，在网站目录中右键压缩包，选择「解压」，点击「确定」
4. 解压完成后，删除压缩包（避免占用空间）。
5. 访问您的域名，程序会跳转到安装界面。按要求填写您的信息即可。

## 5. 安装完成
- **前台访问**：`http://您的域名/`
- **后台访问**：`http://您的域名/admin`

# EMSHOP 系统特殊配置教程
## 1. 如何修改后台登录入口的路径
系统**不支持直接修改 admin 目录名**来变更后台访问路径，此操作会引发各类异常问题。如需修改后台登录入口路径，请按以下步骤操作：

### 配置步骤
1. 打开网站根目录下的 `config.php` 文件；
2. 在文件**末尾**添加如下配置代码（若已存在该配置，直接修改引号内内容即可）：
   ```php
   const ADMIN_PATH_CODE = 'emshop'; # 引号中的内容可自定义
   ```
3. 将 `emshop` 替换为 **8-16位的字母/数字组合**，**禁止包含特殊字符**；
4. 配置完成后，后台登录入口将变更为：
   ```
   http://baidu.com/admin/account.php?action=signin&s=自定义的字符
   ```
   示例（替换为 `8888emshop8888`）：
   ```
   http://baidu.com/admin/account.php?action=signin&s=8888emshop8888
   ```

## 2. 如何开启演示站模式
演示站模式专为展示系统功能设计，开启后会禁止所有数据修改操作，保障演示环境的数据安全。

### 开启方法
1. 打开网站根目录下的 `config.php` 文件；
2. 在文件**末尾**添加以下代码：
   ```php
   const DEMO_MODE = true;
   ```

### 注意事项
- 开启后，程序会拦截所有数据变更操作（包括新增、编辑、删除、修改配置等）；
- 如需恢复正常使用，删除该行代码或将 `true` 改为 `false` 即可：
  ```php
  const DEMO_MODE = false;
  ```
- 此模式仅适用于演示场景，**禁止在生产环境中启用**。

### 总结
1. 修改后台登录路径需通过 `config.php` 配置 `ADMIN_PATH_CODE` 常量，而非直接改目录名；
2. 演示站模式通过 `DEMO_MODE` 常量控制，开启后会锁定所有数据修改操作，仅用于展示。
